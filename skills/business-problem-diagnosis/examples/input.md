# 示例输入｜企业问题诊断 Skill

> 这个文件用于演示如何调用「企业问题诊断 Skill」。

---

## 使用目标

请使用「企业问题诊断 Skill」，基于下面这个问题，判断它属于 A / B / C 哪一类，并给出 72 小时行动建议。

---

## 输入信息

```text
问题描述：
我最近一直在做 AI 工作流、GitHub 样板库和悦达智科公开入口，但悦达传媒官网还没做好，Codex 生成的视觉很难看。我不知道应该继续死磕官网，还是先沉淀 Skill 和公开样板库。

问题背景：
我想扩大影响力，也想未来变现。但现在事情很多，官网、公众号、GitHub、Codex、Skill、直播、内容生产都在推进，容易乱。

涉及对象：
我自己、悦达传媒、悦达智科、潜在企业客户、未来合作伙伴

当前影响：
时间精力、品牌可信度、获客路径、情绪压力、公开展示资产

已经做过什么：
已经上线 GitHub 个人主页、悦达智科官网入口、公开样板库，并完成第一个公众号资料包生成 Skill。

目前最担心什么：
怕自己做了很多事情，但没有形成真正能带来客户信任和商业转化的公开资产。

期望结果：
判断下一步最值得做什么，哪些事该先做，哪些事该暂缓。

可用资源：
ChatGPT、Codex、GitHub、公众号、视频号、已有内容经验、企业服务经验。

必须遵守的限制：
不要一次性铺太多线，不要把时间耗在低质量视觉反复修改上，不要公开客户隐私和商业敏感信息。
```

---

## 期望输出

请输出以下内容：

```text
01_问题复述
02_关键信息缺口
03_问题类型判断
04_A/B/C 类问题分级
05_影响维度评估
06_关注圈 / 影响圈判断
07_根因假设
08_下一步 72 小时行动建议
09_不建议立刻做什么
10_风险提醒
11_是否建议进入深度诊断
12_机器可读 JSON
```

---

## 输出要求

- 区分事实、判断和情绪。
- 不要把所有问题都夸大成 A 类问题。
- 不要只给空泛建议，要给 72 小时内能执行的动作。
- 要说明为什么这样分级。
- 要判断哪些部分属于影响圈，哪些只是关注圈。
- 不要替代法律、财务、投资等专业判断。
- 结尾要有明确下一步。
