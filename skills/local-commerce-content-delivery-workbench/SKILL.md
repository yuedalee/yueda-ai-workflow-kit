# 本地商品内容与发货工作台设计 Skill

> Public Version v0.1  
> 适用场景：帮助一人公司、软件/插件卖家、虚拟商品卖家，把商品内容生产、教程视频处理和半自动发货整理成可落地的本地工作台方案。

---

## 1. Skill 定位

本 Skill 用于把一个模糊的“我想自动化卖东西、做图、做视频、发货”的需求，转化为一套可执行、可验收、低成本、低风险的本地工作台设计。

它不是提示词合集，也不是直接承诺全自动平台操作。

核心目标：

> 把商品资料、素材生产、视频教程、发货话术和发货记录整理成一条可复用的半自动生产线。

---

## 2. 适用场景

适合用于：

1. 软件、插件、教程、资料包等虚拟商品销售；
2. 拼多多、淘宝、闲鱼等平台的商品素材准备；
3. 商品主图、轮播图、详情页批量生成；
4. 软件/插件教程视频录屏后处理；
5. 下载链接、激活码、卡密、安装教程的半自动发货；
6. 一人公司希望降低重复劳动但不想依赖付费 SaaS。

不适合用于：

1. 绕过平台验证码、风控或规则；
2. 刷单、刷评、刷曝光；
3. 大规模骚扰式私信；
4. 自动承诺售后或自动处理退款；
5. 需要法律、财务、平台合规结论的高风险判断。

## 5. 执行流程

### Step 1：先判断真实业务目标

区分用户到底要的是：

```text
商品素材生产
教程视频处理
自动发货
平台操作自动化
客服问答
全流程中台
```

默认不要把需求升级成复杂中台。

---

### Step 2：先压缩 V1 范围

优先保留：

```text
商品资料结构化
标准文件输出
半自动发货
日志记录
人工确认
```

优先排除：

```text
自动登录平台
绕过验证码
复杂 Agent 编排
付费 API 依赖
全自动客服
```

---

### Step 3：设计本地确定性生产线

按以下顺序设计：

```text
商品资料
→ 图片和详情页
→ 发货话术与日志
→ 视频处理
→ 简单界面
→ 平台辅助自动化
```

---

### Step 4：给 Codex 可执行任务

任务必须小到可验收。

每个任务必须包含：

```text
目标
输入
输出
修改文件
运行命令
验收标准
失败条件
```

---

### Step 5：给出一票否决项

任何方案如果出现以下问题，必须判定为不可用：

```text
必须订阅才能跑核心流程
没有标准文件输出
没有日志
重复订单会重复发货
图片或视频不能直接使用
出错无法定位
平台自动化绕过风控
```

---

## 6. 输出质量标准

合格输出必须满足：

1. 先给结论，再给步骤；
2. 明确哪些做、哪些不做；
3. 不堆工具名；
4. 不承诺不稳定的全自动平台操作；
5. 每一步都有输入、输出、验收标准；
6. 能指导 Codex 分阶段构建；
7. 能防止“做出来但不能用”。

---

## 7. 风险提醒

必须提醒：

1. 平台后台自动化可能受验证码、风控、页面改版影响；
2. 虚拟商品发货必须防止重复发货和错发；
3. 商品图和详情页必须人工审核，不得自动发布；
4. 涉及客户隐私、订单数据、账号信息时不能写入公开仓库；
5. 不应把 API Key、Cookie、Token、Webhook 提交到 GitHub。
