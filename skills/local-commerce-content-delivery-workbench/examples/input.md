# 示例输入｜本地商品内容与发货工作台设计 Skill

```text
任务名称：设计一个一人公司使用的本地商品内容与发货工作台

任务背景：
我需要卖一些软件、插件、教程、资料包类商品。现在最大的问题是商品图、轮播图、详情页、教程视频和发货都很耗时间。之前听过很多自动化方案，但担心 Codex 搭出来只是能跑，最后不能拿回去用。

目标用户：
一人公司、软件/插件卖家、虚拟商品卖家。

销售平台：
拼多多、淘宝、闲鱼。

核心目标：
1. 自动或半自动生成商品主图、轮播图、详情页；
2. 处理软件/插件教程录屏，生成横版和竖版教程视频；
3. 自动生成发货话术；
4. 管理卡密/激活码/下载链接；
5. 记录发货日志，防止漏发和重复发货。

已有素材：
- 软件截图；
- 浏览器插件录屏；
- 商品卖点文案；
- 下载链接；
- 安装教程；
- 部分卡密。

希望避免：
- 一上来做复杂中台；
- 依赖必须付费订阅；
- 自动登录平台导致风控；
- 做出一堆功能但输出文件不能直接用；
- 没有日志，出错无法追查。

希望输出：
1. 顶层设计；
2. 每一步怎么落地；
3. Codex 任务拆解；
4. 最终验收标准；
5. 风险边界。
```
