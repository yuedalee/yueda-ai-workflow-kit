# 示例输出骨架｜本地商品内容与发货工作台设计 Skill

```text
01_核心判断

不要先做全自动电商机器人。
先做一个本地商品内容与发货工作台。
它的价值不是替代你，而是把 80% 重复劳动变成标准文件输出，把 20% 高风险动作留给人工确认。

02_V1 系统范围

V1 只做三件事：
1. 商品图和详情页模板化生成；
2. 教程录屏半自动处理；
3. 发货话术、卡密、日志半自动管理。

03_不做事项

不做自动登录、绕过验证码、自动发布商品、自动发送买家消息、刷单刷评、必须付费 API。

04_输入标准

每个商品维护一个 product.yaml，包含商品名、平台、价格、目标用户、痛点、卖点、素材路径、发货内容、FAQ。

05_输出标准

每个商品生成 output 文件夹，包含 images、detail、video、delivery 四类文件。

06_技术栈

Python、SQLite、YAML、Jinja2、Playwright、Pillow、FFmpeg、auto-editor、faster-whisper。

07_落地步骤

Task 01：项目骨架和 CLI
Task 02：商品资料读取和校验
Task 03：输出目录和日志
Task 04：商品主图和轮播图生成
Task 05：详情页生成
Task 06：订单文本解析
Task 07：发货话术和日志
Task 08：卡密池管理
Task 09：视频检查
Task 10：剪静音、字幕和导出视频
Task 11：简单本地界面

08_验收标准

能生成可上传图片、750px 详情长图、横竖版视频、SRT 字幕、可复制发货话术、发货日志和重复订单提醒。

09_风险提醒

平台自动化最后做；凡是涉及发布、发货、改价、买家消息的动作都必须保留人工确认。
```
